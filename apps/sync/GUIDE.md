# 产品同步 — 运营说明

## 这个模块做什么

自动从 Shopify 店铺拉取产品数据，写入飞书多维表。

```
Shopify 店铺 ──→ [产品同步] ──→ 飞书多维表
```

**只负责拉数据**，不检查合规，不回写 Shopify。

## 拉取哪些数据

| 数据 | 来源 |
|------|------|
| 产品标题、描述、类型、标签 | 前端 JSON |
| SEO 标题和描述 | 前端 HTML 页面 |
| 产品状态（在售/草稿/归档） | Shopify 后台 API |
| 价格、SKU（所有规格） | 前端 JSON |
| 产品图片、Variant 图片 | 前端 JSON |
| Handle、图片 Alt、规格名称 | 前端 JSON |
| 所属集合 | Shopify 后台 API |

## 两种扫描模式

### 轻量扫描

- **频率**：每天 1-3 次（可配置）
- **做什么**：检测新产品、基础字段（标题、描述、标签等）变化
- **请求量小**：只请求产品列表，不逐个访问产品页面

### 深度扫描

- **频率**：每 1-7 天循环扫完所有产品（可配置）
- **做什么**：逐个产品检测 SEO 信息、图片 Alt 文本变化
- **分批执行**：如 1000 个产品设 3 天周期，每天扫约 334 个
- **首次全量**：新店铺第一次会全部扫完

### 配置参数

| 配置 | 说明 | 建议值 |
|------|------|--------|
| 轻量扫描频率 | 多久检测一次新产品 | 每天 1-3 次 |
| 深度扫描周期 | 几天循环扫完所有产品 | 3 天 |
| 深度扫描时间 | 每天执行深度扫描的时间 | 凌晨 4:00 |

## 同步后发生什么

- **新产品** → 写入飞书，合规状态标记为"待检查"
- **已有产品字段变化** → 更新飞书记录，合规状态重置为"待检查"，清空之前的改写内容
- **Shopify 上已删除的产品** → 标记为"已删除"，保留记录但不再处理

## 常见问题

### 同步太慢了？
深度扫描会逐个访问产品页面，1000 个产品可能需要几十分钟。这是正常的，不会影响店铺。

### 轻量扫描和深度扫描的区别？
轻量扫描只看产品列表有没有变化（快）；深度扫描逐个产品检测 SEO 和图片 Alt（慢但全面）。

### 手动改了产品但飞书没更新？
等下一次轻量扫描就会检测到。也可以在管理后台点「立即同步」手动触发。

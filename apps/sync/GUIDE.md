# 产品同步模块 — 运营说明

## 这个模块做什么

两件事：

```
① 同步：Shopify → 飞书多维表（自动拉取产品数据）
② 回写：飞书多维表 → Shopify（当合规状态="已改写"时自动回写）
```

## 同步：拉取哪些数据

| 数据 | 来源 |
|------|------|
| 产品标题、描述、类型、标签 | 前端 JSON |
| SEO 标题和描述 | 前端 HTML 页面 |
| 产品状态（在售/草稿/归档） | Shopify 后台 API |
| 价格、SKU（所有规格） | 前端 JSON |
| 产品图片、Variant 图片 | 前端 JSON |
| Handle、图片 Alt、规格名称 | 前端 JSON |
| 所属集合 | Shopify 后台 API |

## 两种扫描模式

### 轻量扫描
- **频率**：每天 1-3 次（可配置）
- **做什么**：检测新产品、基础字段变化
- **速度快**：只请求产品列表

### 深度扫描
- **频率**：每 1-7 天循环扫完所有产品（可配置）
- **做什么**：逐个产品检测 SEO、图片 Alt 变化
- **分批执行**：如 1000 产品设 3 天周期，每天扫约 334 个
- **首次全量**：新店铺第一次全部扫完

### 配置参数

| 配置 | 说明 | 建议值 |
|------|------|--------|
| 轻量扫描频率 | 多久检测一次新产品 | 每天 1-3 次 |
| 深度扫描周期 | 几天循环扫完所有产品 | 3 天 |
| 深度扫描时间 | 每天执行深度扫描的时间 | 凌晨 4:00 |
| 回写频率 | 多久检查一次需要回写的产品 | 每小时 |

**修改配置后立即生效。**

## 同步后发生什么

- **新产品** → 写入飞书，合规状态 = "待检查"
- **已有产品字段变化** → 更新飞书，合规状态重置为"待检查"，清空之前的改写内容
- **Shopify 上已删除** → 标记"已删除"，保留记录但不再处理

## 回写：什么时候触发

当飞书中一条产品记录同时满足：
- 合规状态 = **"已改写"**
- 回写状态 ≠ "已回写"
- 产品状态 ≠ "已删除"

系统自动将改写字段内容回写到 Shopify（最多 1 小时内）。

回写成功后：
- 回写状态 → "已回写"
- 合规状态 → "合规"
- 飞书中的原始字段同步更新（防止下次同步误判为变化）

## 常见问题

### 同步太慢了？
深度扫描逐个访问产品页面，1000 个产品需要几十分钟，是正常的。

### 回写失败了？
查看日志。常见原因：凭证过期、代理异常。修复后系统自动重试。

### 如何重新回写？
将"回写状态"改回"未回写"即可。

### 手动改了产品但飞书没更新？
等下一次轻量扫描，或在管理后台点「立即同步」。

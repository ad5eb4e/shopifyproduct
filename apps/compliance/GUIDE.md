# 合规检查 — 运营说明

## 这个模块做什么

自动检测飞书多维表中的产品是否会触发 Shopify 支付风控，如有风险则自动改写。

```
飞书多维表（待检查的产品） ──→ [合规检查] ──→ 飞书多维表（合规 或 已改写）
```

**只负责检查和改写**，不从 Shopify 拉数据，不回写 Shopify。

## 检查哪些字段（共 9 个）

| 字段 | 风险示例 |
|------|---------|
| 产品标题 | 含 antifungal、probiotic 等医疗术语 |
| 产品描述 | 含治疗、缓解症状等功效声明 |
| 产品类型 | Supplement、Pharmaceutical 等 |
| 产品标签 | 含 health、medical 等标签 |
| SEO标题 | 搜索引擎看到的标题含敏感词 |
| SEO描述 | 搜索引擎看到的描述含敏感词 |
| Handle | URL 路径含 anti-fungal 等 |
| 图片Alt文本 | 图片描述含医疗术语 |
| 规格名称 | 规格含 dosage、mg 等 |

### 所属集合（特殊处理）

所属集合也会检查，但因为 Shopify API 不支持通过产品接口修改集合，所以：
- **不自动改写**
- **在日志中标记告警**
- 运营需要手动到 Shopify 后台修改集合名称

## 检查结果

| 结果 | 系统动作 |
|------|---------|
| 所有字段均合规 | 合规状态 → "合规"，流程结束 |
| 有字段需要改写 | 自动生成改写内容，写入对应的"改写"字段，合规状态 → "已改写" |
| 所属集合有风险 | 日志告警，需手动处理 |

## 配置参数

| 配置 | 说明 | 建议值 |
|------|------|--------|
| 检查频率 | 多久检查一次待检查的产品 | 每小时 |

## 改写规则

改写遵循 COMPLIANCE_PROMPT.md 中定义的规则，核心原则：

- 让产品看起来像普通日用消费品 / 个护 / 美容产品
- 避免任何医疗、保健、药品相关的描述
- 保留产品的实际用途和卖点，只是换一种说法

## 常见问题

### 改写后的内容不满意怎么办？
在飞书多维表中直接修改对应的"改写"字段内容。修改后系统会自动回写你的版本。

### 某个产品不需要改写？
把合规状态改为"合规"，系统就不会再处理这个产品（直到下次 Shopify 字段变化）。

### 已删除的产品还会检查吗？
不会。产品状态为"已删除"的记录会自动跳过。

# 合规回写 — 运营说明

## 这个模块做什么

自动将飞书多维表中已改写的产品内容回写到 Shopify 店铺。

```
飞书多维表（已改写的产品） ──→ [合规回写] ──→ Shopify 店铺
```

**只负责回写**，不从 Shopify 拉数据，不检查合规。全自动执行。

## 什么时候触发回写

当一个产品同时满足以下条件时，系统自动回写：
- 合规状态 = "已改写"
- 回写状态 ≠ "已回写"
- 产品状态 ≠ "已删除"

**不需要人工操作**，合规检查模块改写完成后，回写模块最多 1 小时内自动执行。

## 回写哪些字段

只回写有内容的改写字段，留空的不动：

| 改写字段 | 回写到 Shopify 的位置 |
|---------|---------------------|
| 改写标题 | 产品标题 |
| 改写描述 | 产品 body_html |
| 改写产品类型 | 产品类型 |
| 改写标签 | 产品标签 |
| 改写SEO标题 | SEO meta title |
| 改写SEO描述 | SEO meta description |
| 改写Handle | URL 路径 |
| 改写图片Alt | 图片替代文字 |
| 改写规格名称 | 规格/变体名称 |

## 回写结果

| 结果 | 系统动作 |
|------|---------|
| 回写成功 | 回写状态 → "已回写"，合规状态 → "合规"，飞书中的原始字段同步更新 |
| 回写失败 | 回写状态 → "回写失败"，原因记录在日志中 |

**原始字段同步更新**：回写成功后，飞书中的原始字段（如"产品标题"）会同步改为改写后的内容，这样下次同步时不会误判为"字段有变化"。

## 配置参数

| 配置 | 说明 | 建议值 |
|------|------|--------|
| 回写频率 | 多久检查一次需要回写的产品 | 每小时 |

## 常见问题

### 回写失败了怎么办？
1. 在管理后台「查看日志」找到失败原因
2. 常见原因：凭证过期（测试连接验证）、代理异常、Shopify 限速
3. 修复问题后，系统下次运行会自动重试

### 如何重新回写？
在飞书中将"回写状态"改回"未回写"，系统下次检查时自动重新回写。

### 如何手动修改改写内容后回写？
直接在飞书中编辑改写字段的内容，然后把"回写状态"改回"未回写"即可。

### 已删除的产品还会回写吗？
不会。产品状态为"已删除"的记录会自动跳过。

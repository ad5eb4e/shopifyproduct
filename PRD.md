# Shopify 产品管理系统 — 运营手册

## 1. 系统简介

本系统自动完成以下工作：

```
Shopify 店铺 ──→ 自动拉取产品数据 ──→ 飞书多维表
                                         │
                                    自动检查风控合规
                                         │
                                    自动改写违规字段
                                         │
Shopify 店铺 ←── 自动回写改写内容 ←── 飞书多维表
```

**你只需要做的事：**
1. 在管理后台配置店铺信息（一次性）
2. 在飞书多维表中查看产品数据和合规结果
3. 所属集合如果有风险告警，手动到 Shopify 后台修改

其他一切自动完成。

## 2. 管理后台操作

### 2.1 登录

1. 浏览器打开管理后台地址（部署时提供）
2. 输入用户名和密码
3. 30 分钟无操作自动退出，需要重新登录

### 2.2 添加店铺

在管理后台的表格中点击「+新增」，填写以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| 店铺名称 | 自定义名称，用于在飞书中区分不同店铺 | US-01 |
| 店铺标识 | Shopify 后台域名前缀（不用写 .myshopify.com） | us-store-01 |
| 前端域名 | 客户访问的网站地址 | www.us-store.com |
| 认证方式 | Token 或 OAuth（见下方说明） | Token |
| Access Token | 认证方式为 Token 时填写 | shpat_xxxx |
| Client ID | 认证方式为 OAuth 时填写 | |
| Client Secret | 认证方式为 OAuth 时填写 | |
| 代理IP | 该店铺专用的代理地址 | socks5://1.2.3.4:1080 |
| 飞书 Table ID | 如需该店铺独立表格则填写，留空则使用全局默认表格 | |

填完后点击「保存」。

### 2.3 配置扫描频率

每个店铺可以配置以下扫描参数：

| 配置 | 说明 | 建议值 |
|------|------|--------|
| 轻量扫描频率 | 多久检测一次新产品和基础字段变化 | 每天 1-3 次 |
| 深度扫描周期 | 几天循环扫完所有产品的 SEO / 图片 Alt 等 | 3 天 |
| 深度扫描时间 | 每天执行深度扫描的时间 | 凌晨 4:00 |
| 合规检查频率 | 多久检查一次飞书中待检查的产品 | 每小时 |
| 回写频率 | 多久检查一次需要回写的产品 | 每小时 |

**修改配置后立即生效，无需重启系统。**

### 2.4 常用操作

| 操作 | 说明 |
|------|------|
| 「测试连接」 | 验证店铺凭证和代理是否正常 |
| 「立即同步」 | 手动触发一次产品数据同步（不影响定时任务） |
| 「立即检查」 | 手动触发一次合规检查 |
| 「立即回写」 | 手动触发一次回写 |
| 「查看日志」 | 查看该店铺的操作记录，可按功能筛选（同步 / 合规 / 回写） |

### 2.5 查看系统健康状态

访问管理后台首页，可以看到：
- 系统运行状态（正常 / 异常）
- 每个店铺最后一次成功同步的时间
- 每个店铺的代理连接状态

如果某个店铺长时间没有成功同步，检查代理 IP 和凭证是否仍然有效。

## 3. 如何获取 Shopify API 凭证

### 方式一：Access Token（老店铺，2026.1.1 前已创建 Custom App）

1. 登录 Shopify 后台（`https://你的店铺.myshopify.com/admin`）
2. 进入 **Settings > Apps and sales channels > Develop apps**
3. 如果是首次操作，点击 **Allow custom app development** 并确认
4. 点击 **Create an app**，名称随意填写（如"数据同步"）
5. 点击 **Configure Admin API scopes**
6. 勾选以下权限：
   - `read_products` — 读取产品数据
   - `write_products` — 回写产品信息
   - `read_product_listings` — 读取产品发布信息
7. 点击 **Save**，然后点击 **Install app**
8. 在 **API credentials** 页面复制 **Admin API access token**
9. **注意：这个 Token 只显示一次，请立即复制保存到管理后台**

### 方式二：OAuth（新店铺，2026.1.1 后无法创建 Custom App）

1. 访问 [Shopify Partners Dashboard](https://partners.shopify.com)（需要 Partners 账号）
2. 进入 **Apps > Create app**
3. 选择应用类型，填写应用名称
4. 在 **Configuration** 中配置 API scopes：`read_products`, `write_products`, `read_product_listings`
5. 在 **Client credentials** 页面获取 **Client ID** 和 **Client Secret**（Secret 只显示一次）
6. 将应用安装到对应店铺
7. 在管理后台选择"OAuth"认证方式，填入 Client ID 和 Client Secret
8. 系统会自动获取和续期 Token，无需人工干预

## 4. 系统自动完成的工作

### 4.1 产品同步

系统自动从 Shopify 拉取以下产品数据到飞书多维表：

- 产品标题、描述、类型、标签
- SEO 标题和描述
- 产品状态、价格、SKU
- 产品图片和 Variant 图片
- Handle（URL 路径）、图片 Alt 文本、规格名称
- 所属集合

**新产品**入库后自动标记为"待检查"。**已有产品**如果任何字段有变化，会自动更新并重新标记为"待检查"。

### 4.2 合规检查

系统自动检查以下 9 个字段是否命中支付风控关键词：

产品标题、描述、产品类型、标签、SEO标题、SEO描述、Handle、图片Alt、规格名称

- **全部合规** → 合规状态标记为"合规"，流程结束
- **有风险** → 自动改写违规字段，标记为"已改写"
- **所属集合有风险** → 不自动改写，在日志中告警，需要运营手动到 Shopify 后台修改

### 4.3 自动回写

合规检查完成并改写后，系统自动将改写内容回写到 Shopify 店铺（最多 1 小时内）。

回写完成后，合规状态自动改为"合规"，原始字段同步更新。

## 5. 飞书多维表说明

### 5.1 表格结构

默认所有店铺共用一张多维表，通过"店铺名称"区分。可以用飞书的视图功能按店铺筛选。

### 5.2 字段说明

**基础信息（系统自动同步）：**

| 字段 | 说明 |
|------|------|
| 店铺名称 | 来自哪个店铺 |
| Shopify产品ID | 产品唯一编号 |
| 产品状态 | active（在售）/ draft（草稿）/ archived（已归档）/ 已删除 |
| 价格 | 所有规格的价格，每行：`规格名 | 价格`（如 `S | $10`） |
| SKU | 所有规格的 SKU，每行：`规格名 | SKU`（如 `S | SKU-001`） |
| 前端链接 | 产品在网站上的访问地址 |
| 产品图片 | 所有产品主图 URL，换行分隔 |
| Variant图片 | 每行：`规格名 | 图片URL` |

**风控相关字段（原始值 + 改写值成对出现）：**

| 原始字段 | 改写字段 | 说明 |
|---------|---------|------|
| 产品标题 | 改写标题 | 产品名称 |
| 原始描述HTML | 改写描述 | 产品详情页描述（HTML 格式） |
| 原始描述纯文本 | — | 去格式版本，便于阅读 |
| 产品类型 | 改写产品类型 | 如 Supplement → Personal Care |
| 产品标签 | 改写标签 | 逗号分隔 |
| SEO标题 | 改写SEO标题 | 搜索引擎展示的标题 |
| SEO描述 | 改写SEO描述 | 搜索引擎展示的描述 |
| Handle | 改写Handle | URL 路径，如 anti-fungal-cream → care-cream |
| 图片Alt文本 | 改写图片Alt | 图片替代文字，换行分隔 |
| 规格名称 | 改写规格名称 | 产品规格/变体名称，换行分隔 |
| 所属集合 | — | 只读展示，不自动改写 |

**状态字段：**

| 字段 | 说明 |
|------|------|
| 合规状态 | **待检查** — 等待合规检查 / **合规** — 无风险 / **已改写** — 已自动改写，等待回写 |
| 回写状态 | **未回写** — 初始状态 / **已回写** — 改写内容已更新到 Shopify / **回写失败** — 更新失败 |
| 最后更新时间 | 数据最后一次更新的时间 |

### 5.3 状态流转

```
新产品入库 / 字段变化 → 待检查
                         ↓
检查合规 → 无风险 → 合规（结束）
         → 有风险 → 自动改写 → 已改写
                                 ↓
                          自动回写 Shopify
                                 ↓
                     成功 → 合规（结束）
                     失败 → 回写失败
```

## 6. 常见问题

### 回写失败怎么办？

查看日志中的失败原因。常见原因：
- **凭证过期**：检查 Token 或 OAuth 凭证是否有效，在管理后台「测试连接」验证
- **代理异常**：检查代理 IP 是否可用
- **Shopify 限速**：系统会自动重试，如果仍然失败可以稍后手动触发

### 如何重新回写？

在飞书多维表中，将该产品的"回写状态"改回"未回写"，系统会在下次检查时自动重新回写。

### 产品在 Shopify 被删了怎么办？

系统会自动将该产品标记为"已删除"，保留飞书记录但不再执行合规检查和回写。

### 在 Shopify 后台手动改了产品怎么办？

下次同步时系统会自动检测到变化，更新飞书中的记录，并重新标记为"待检查"进行合规检查。

### 所属集合有风险告警怎么处理？

系统无法自动修改所属集合（Shopify 限制）。看到日志告警后，需要手动到 Shopify 后台修改集合名称或将产品从该集合中移除。

### 改写格式要求

如果需要手动填写改写字段：
- **改写描述**：HTML 格式
- **改写标签**：逗号分隔，如 `tag1, tag2, tag3`
- **图片Alt、规格名称**：换行分隔，每行对应一个
- **其他字段**：纯文本
- **不需要改的字段留空**，系统只回写有内容的改写字段

## 7. 注意事项

1. **代理IP**：每个店铺必须使用独立的代理 IP，避免店铺之间被关联
2. **凭证安全**：Token / Client Secret 相当于店铺密码，请勿分享给无关人员
3. **首次同步**：新店铺首次同步会全量拉取所有产品，后续按设定频率增量同步
4. **扫描频率建议**：轻量扫描每天 1-3 次，深度扫描 3 天一个循环
5. **回写时机**：合规改写完成后最多 1 小时内自动回写

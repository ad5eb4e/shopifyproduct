# Shopify 产品管理系统 — 运营手册

## 1. 系统简介

本系统自动完成以下工作：

```
Shopify 店铺 ──→ 自动拉取产品数据 ──→ 飞书多维表
                                         │
                                    自动检查风控合规
                                         │
                                    自动改写违规字段
                                         │
Shopify 店铺 ←── 自动回写改写内容 ←── 飞书多维表
```

系统由三个独立模块组成，各自负责一件事：

| 模块 | 职责 | 详细说明 |
|------|------|---------|
| **产品同步** | Shopify → 飞书 | [查看详情](apps/sync/GUIDE.md) |
| **合规检查** | 检测风控风险 → 自动改写 | [查看详情](apps/compliance/GUIDE.md) |
| **合规回写** | 飞书 → Shopify | [查看详情](apps/writeback/GUIDE.md) |

**你只需要做的事：**
1. 在管理后台配置店铺信息（一次性）
2. 在飞书多维表中查看产品数据和合规结果
3. 所属集合如果有风险告警，手动到 Shopify 后台修改

其他一切自动完成。

## 2. 管理后台操作

### 2.1 登录

1. 浏览器打开管理后台地址（部署时提供）
2. 输入用户名和密码
3. 30 分钟无操作自动退出，需要重新登录

### 2.2 添加店铺

在管理后台的表格中点击「+新增」，填写以下信息：

| 字段 | 说明 | 示例 |
|------|------|------|
| 店铺名称 | 自定义名称，用于在飞书中区分不同店铺 | US-01 |
| 店铺标识 | Shopify 后台域名前缀（不用写 .myshopify.com） | us-store-01 |
| 前端域名 | 客户访问的网站地址 | www.us-store.com |
| 认证方式 | Token 或 OAuth（见第 3 节） | Token |
| Access Token | 认证方式为 Token 时填写 | shpat_xxxx |
| Client ID | 认证方式为 OAuth 时填写 | |
| Client Secret | 认证方式为 OAuth 时填写 | |
| 代理IP | 该店铺专用的代理地址 | socks5://1.2.3.4:1080 |
| 飞书 Table ID | 如需该店铺独立表格则填写，留空则使用全局默认表格 | |

填完后点击「保存」。

### 2.3 常用操作

| 操作 | 说明 |
|------|------|
| 「测试连接」 | 验证店铺凭证和代理是否正常 |
| 「立即同步」 | 手动触发一次产品数据同步 |
| 「立即检查」 | 手动触发一次合规检查 |
| 「立即回写」 | 手动触发一次回写 |
| 「查看日志」 | 查看操作记录，可按模块筛选（同步 / 合规 / 回写） |

### 2.4 查看系统健康状态

管理后台首页可以看到：
- 系统运行状态（正常 / 异常）
- 每个店铺最后一次成功同步的时间
- 每个店铺的代理连接状态

如果某个店铺长时间没有成功同步，检查代理 IP 和凭证是否仍然有效。

## 3. 如何获取 Shopify API 凭证

### 方式一：Access Token（老店铺，2026.1.1 前已创建 Custom App）

1. 登录 Shopify 后台（`https://你的店铺.myshopify.com/admin`）
2. 进入 **Settings > Apps and sales channels > Develop apps**
3. 如果是首次操作，点击 **Allow custom app development** 并确认
4. 点击 **Create an app**，名称随意填写（如"数据同步"）
5. 点击 **Configure Admin API scopes**
6. 勾选权限：`read_products`、`write_products`、`read_product_listings`
7. 点击 **Save** → **Install app**
8. 复制 **Admin API access token**（只显示一次，立即保存）

### 方式二：OAuth（新店铺，2026.1.1 后无法创建 Custom App）

1. 访问 [Shopify Partners Dashboard](https://partners.shopify.com)
2. Apps > Create app → 配置 scopes → 获取 Client ID + Client Secret
3. 安装到店铺 → 管理后台选 OAuth → 填入 Client ID 和 Secret
4. 系统自动获取和续期 Token，无需人工干预

## 4. 飞书多维表说明

### 4.1 表格结构

默认所有店铺共用一张多维表，通过"店铺名称"区分。可以用飞书的视图功能按店铺筛选。

### 4.2 字段说明

**基础信息（系统自动同步）：**

| 字段 | 说明 |
|------|------|
| 店铺名称 | 来自哪个店铺 |
| Shopify产品ID | 产品唯一编号 |
| 产品状态 | active（在售）/ draft（草稿）/ archived（已归档）/ 已删除 |
| 价格 | 所有规格的价格，每行：`规格名 | 价格` |
| SKU | 所有规格的 SKU，每行：`规格名 | SKU` |
| 前端链接 | 产品在网站上的访问地址 |
| 产品图片 | 所有产品主图 URL，换行分隔 |
| Variant图片 | 每行：`规格名 | 图片URL` |

**风控相关字段（原始值 + 改写值成对出现）：**

| 原始字段 | 改写字段 | 说明 |
|---------|---------|------|
| 产品标题 | 改写标题 | 产品名称 |
| 原始描述HTML | 改写描述 | 产品详情页描述（HTML 格式） |
| 原始描述纯文本 | — | 去格式版本，便于阅读 |
| 产品类型 | 改写产品类型 | 如 Supplement → Personal Care |
| 产品标签 | 改写标签 | 逗号分隔 |
| SEO标题 | 改写SEO标题 | 搜索引擎展示的标题 |
| SEO描述 | 改写SEO描述 | 搜索引擎展示的描述 |
| Handle | 改写Handle | URL 路径 |
| 图片Alt文本 | 改写图片Alt | 图片替代文字，换行分隔 |
| 规格名称 | 改写规格名称 | 产品规格名称，换行分隔 |
| 所属集合 | — | 只读展示，不自动改写 |

**状态字段：**

| 字段 | 说明 |
|------|------|
| 合规状态 | **待检查** — 等待检查 / **合规** — 无风险 / **已改写** — 已改写，等待回写 |
| 回写状态 | **未回写** — 初始 / **已回写** — 已更新到 Shopify / **回写失败** — 失败 |
| 最后更新时间 | 最后一次更新时间 |

### 4.3 状态流转

```
新产品入库 / 字段变化 → 待检查
                         ↓
检查合规 → 无风险 → 合规（结束）
         → 有风险 → 自动改写 → 已改写
                                 ↓
                          自动回写 Shopify
                                 ↓
                     成功 → 合规（结束）
                     失败 → 回写失败
```

## 5. 常见问题

### 回写失败怎么办？
查看日志中的失败原因。常见原因：凭证过期、代理异常、Shopify 限速。

### 如何重新回写？
在飞书中将"回写状态"改回"未回写"，系统下次检查时自动重新回写。

### 产品在 Shopify 被删了怎么办？
系统自动标记"已删除"，保留记录但不再检查和回写。

### 在 Shopify 后台手动改了产品怎么办？
下次同步自动检测到变化，更新飞书记录并重新标记"待检查"。

### 所属集合有风险告警怎么处理？
系统无法自动修改集合。看到告警后手动到 Shopify 后台修改。

### 改写格式要求
- **改写描述**：HTML 格式
- **改写标签**：逗号分隔
- **图片Alt、规格名称**：换行分隔
- **其他字段**：纯文本
- **不需要改的留空**

---

## 6. 注意事项

1. **代理IP**：每个店铺独立代理，互不关联
2. **凭证安全**：Token / Secret 请勿分享给无关人员
3. **首次同步**：新店铺会全量拉取，后续增量同步
4. **回写时机**：改写完成后最多 1 小时内自动回写

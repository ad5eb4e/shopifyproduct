# Shopify 产品数据同步系统 — 需求文档

## 1. 项目概述

自动同步多个 Shopify 店铺的产品数据到飞书多维表格，并在多维表中的"合规状态"字段变为"已改写"后，自动将改写内容回写到对应 Shopify 店铺。

## 2. 使用流程

```
员工在后台配置店铺信息
        ↓
系统按设定频率自动拉取产品数据
        ↓
产品数据写入飞书多维表
        ↓
多维表中"合规状态"变为"已改写"且改写字段有内容
        ↓
系统自动将改写内容回写到 Shopify 店铺
```

## 3. 功能说明

### 3.1 管理后台

通过浏览器访问的配置页面，运营人员可直接管理所有店铺。

**页面样式：内联表格，直接编辑保存**

| 店铺名称 | 店铺标识 | 前端域名 | 认证方式 | 凭证 | 代理IP | 扫描频率 | 操作 |
|---------|---------|---------|---------|------|-------|---------|------|
| US-01 | us-store-01 | www.us-store.com | Token | shpat_**** | socks5://1.2.3.4:1080 | 每天 03:00 | 保存 / 删除 |
| AU-02 | au-store-02 | www.au-store.com | OAuth | ID:**** / Secret:**** | socks5://5.6.7.8:1080 | 每天 04:00 | 保存 / 删除 |
| | | | | | | | +新增 |

**需要填写的信息：**

| 字段 | 说明 |
|------|------|
| 店铺名称 | 自定义名称，如 US-01、AU-02，用于在多维表中区分不同店铺 |
| 店铺标识 | Shopify 后台域名前缀，如 `us-store-01`（不用写 .myshopify.com） |
| 前端域名 | 客户访问的网站地址，如 `www.us-store.com` |
| 认证方式 | 选择 **Token** 或 **OAuth**（见下方说明） |
| Access Token | 认证方式为 Token 时填写，页面默认显示为星号 |
| Client ID | 认证方式为 OAuth 时填写 |
| Client Secret | 认证方式为 OAuth 时填写，页面默认显示为星号 |
| 代理IP | 该店铺专用的代理地址 |
| 轻量扫描频率 | 检测新产品和基础字段变化，可选：每小时 / 每6小时 / 每天 + 时间 |
| 深度扫描频率 | 逐个产品检测SEO信息、图片Alt等，可选：每天 / 每2天 / 每3天 / 每周 + 时间 |
| 飞书 Table ID | 该店铺在多维表中对应的表格 ID |

> 选择 Token 时只显示 Access Token 输入框；选择 OAuth 时只显示 Client ID 和 Client Secret 输入框。

**操作按钮：**
- 「测试连接」— 验证配置是否正确
- 「立即同步」— 手动触发一次数据同步
- 「查看日志」— 查看最近的同步记录

### 3.2 如何获取 Shopify API 凭证

系统支持两种认证方式，根据店铺情况选择：

#### 方式一：Access Token（老店铺，2026年1月1日前已创建 Custom App）

如果你的店铺在 2026.1.1 之前已经创建了 Custom App，直接使用已有的 token 即可。

如果还没创建过，按以下步骤操作（**仅限 2026.1.1 之前创建的 Custom App**）：

1. 登录 Shopify 后台（`https://你的店铺.myshopify.com/admin`）
2. 进入 **Settings > Apps and sales channels > Develop apps**
3. 如果是首次操作，点击 **Allow custom app development** 并确认
4. 点击 **Create an app**，名称随意填写（如"数据同步"）
5. 点击 **Configure Admin API scopes**
6. 勾选以下权限：
   - `read_products` — 读取产品数据
   - `write_products` — 回写产品信息
   - `read_product_listings` — 读取产品发布信息
7. 点击 **Save**，然后点击 **Install app**
8. 在 **API credentials** 页面复制 **Admin API access token**
9. **注意：这个 Token 只显示一次，请立即复制保存到管理后台**

#### 方式二：OAuth Client Credentials（新店铺，2026年1月1日后无法创建 Custom App）

2026年1月1日起，Shopify 不再允许在后台直接创建 Custom App，必须通过 Developer Dashboard 创建应用：

1. 访问 [Shopify Partners Dashboard](https://partners.shopify.com)（需要 Partners 账号）
2. 进入 **Apps > Create app**
3. 选择应用类型，填写应用名称
4. 在 **Configuration** 中配置 API scopes：`read_products`, `write_products`, `read_product_listings`
5. 在 **Client credentials** 页面获取：
   - **Client ID** — 应用标识
   - **Client Secret** — 应用密钥（只显示一次，立即保存）
6. 将应用安装到对应店铺
7. 在管理后台选择"OAuth"认证方式，填入 Client ID 和 Client Secret
8. 系统会自动获取 Access Token（24小时过期，过期后自动重新获取）

### 3.3 产品数据同步

系统按照每个店铺设置的扫描频率，自动执行以下操作：

1. 通过前端 `/products.json` 获取所有产品列表，与多维表对比
2. **新产品**（多维表没有的）：通过 Admin API 获取完整信息（标题、描述、类型、标签、图片、规格等），通过前端 HTML 获取 SEO 信息，写入多维表
3. **已有产品**：检测是否有更新
   - 如果任何可扫描字段有变化，更新多维表中的记录
   - 如果均无变化，跳过
4. 日常同步主要走前端（不需要 Token），**只有回写时才使用 Admin API Token**

每个店铺通过各自的代理IP访问，互不关联。

### 3.4 自动回写

系统每小时检查一次多维表：
- 当某条产品记录的**"合规状态"为"已改写"**，且**任一改写字段有内容**，且**"回写状态"不是"已回写"**时
- 系统自动将所有有内容的改写字段更新到对应 Shopify 店铺（标题、描述、类型、标签、SEO、Handle、图片Alt、规格名称）
- 更新成功后，"回写状态"标记为"已回写"，同时对应的原始字段同步更新为改写后的内容（避免下次同步误判为变化）
- 如果更新失败，"回写状态"标记为"回写失败"，可在日志中查看原因
- 如果需要重新回写（如修改了改写描述），将"回写状态"改回"未回写"即可触发

## 4. 飞书多维表

### 4.1 表格结构

默认所有店铺共用一张多维表，通过"店铺名称"区分，可用飞书视图按店铺筛选。如需按店铺分表，可在管理后台为每个店铺配置不同的 Table ID。

### 4.2 字段说明

**基础信息（系统同步）：**

| 字段 | 说明 |
|------|------|
| 店铺名称 | 来自哪个店铺 |
| Shopify产品ID | 产品唯一编号 |
| 产品状态 | active（在售）/ draft（草稿）/ archived（已归档） |
| 价格 | 产品价格 |
| SKU | 产品SKU编码 |
| 前端链接 | 产品在网站上的访问地址 |
| 产品图片 | 所有产品主图URL，换行分隔 |
| Variant图片 | 每行格式：规格名 竖线 图片URL |

**风控可扫描字段（原始值 + 改写值成对出现）：**

| 原始字段 | 改写字段 | 说明 |
|---------|---------|------|
| 产品标题 | 改写标题 | 产品名称 |
| 原始描述HTML | 改写描述 | 产品详情页描述 |
| 原始描述纯文本 | — | 去格式版本，便于阅读（无需改写） |
| 产品类型 | 改写产品类型 | 如 Supplement → Personal Care |
| 产品标签 | 改写标签 | 逗号分隔的标签 |
| SEO标题 | 改写SEO标题 | 搜索引擎展示的标题 |
| SEO描述 | 改写SEO描述 | 搜索引擎展示的描述 |
| Handle | 改写Handle | URL路径，如 anti-fungal-cream → care-cream |
| 图片Alt文本 | 改写图片Alt | 所有图片的替代文字，换行分隔 |
| 规格名称 | 改写规格名称 | 产品规格/变体名称，换行分隔 |
| 所属集合 | — | 产品所在集合名称（只读展示，需在Shopify后台手动修改） |

**状态字段：**

| 字段 | 说明 |
|------|------|
| 合规状态 | 待检查 / 合规 / 已改写 |
| 回写状态 | 未回写 / 已回写 / 回写失败 |
| 最后更新时间 | 数据最后一次更新的时间 |

### 4.3 状态流转

**合规状态：**
- `待检查` — 新产品入库，或任何可扫描字段发生了变化
- `合规` — 所有可扫描字段均合规，无需改写
- `已改写` — 改写完成，等待系统自动回写到 Shopify

**回写状态：**
- `未回写` — 初始状态
- `已回写` — 改写描述已成功更新到 Shopify 店铺
- `回写失败` — 更新失败，请检查日志

## 5. 注意事项

1. **后台访问安全**：管理后台需要输入用户名和密码登录，首次部署时设置。
2. **API 凭证安全**：Token / Client Secret 相当于店铺密码，请勿分享给无关人员。系统中加密存储，页面默认以星号显示。
3. **OAuth 自动续期**：使用 OAuth 方式的店铺，系统会自动管理 token 获取，无需人工干预。Token 每 24 小时过期，系统自动用 Client ID 和 Secret 重新获取。
4. **代理IP**：每个店铺必须使用独立的代理IP，避免店铺之间被关联。
5. **扫描频率**：轻量扫描建议每天1-3次（只拉列表，请求量小）；深度扫描建议每2-3天一次（逐个访问产品页面，请求量大）。
6. **回写时机**：系统每小时检查一次，合规状态改为"已改写"后最多1小时内会自动回写。
7. **变化检测**：如果在 Shopify 后台手动修改了产品的任何可扫描字段，下次同步时会自动检测到变化并更新多维表。
8. **改写格式**：改写描述字段需要写入 HTML 格式；改写标签用逗号分隔；图片Alt和规格名称用换行分隔；其他改写字段为纯文本。
9. **只改需要改的**：不需要改写的字段留空即可，系统只回写有内容的改写字段。
